nodejs与commonjs之间的关系

nodejs是一种服务器语言。
commonjs是nodejs的规范。

commonjs规范的意义

虽然nodejs的语法层面是用js来写的，但是js的标准ECMAScript存在太多的缺陷，它只是规范了javascript中的词法、类型、表达式、方法、对象等基本要素，所以javascript本身是存在的缺点：

  1. 没有模块系统（不方便扩展）
  2. 标准库较少，对于文件系统，I/O流等常见的需求没有标准的API
  3. 没有标准接口，缺少如：WEB胳器，数据库的统一接口

而nodejs作为一门后台语言，它必须有自己的很多基本功能，而这些功能需要有人来规范有，commonjs就是起的这样一个规范，它规范了：

  1. javascript必须像后台语言一样具有拥有开发大型web服务器的有能力
  2. commonjs覆盖了：二进制、字符集编码、I/O流、web服务器…..
  3. 规范了模块与模块之间的引用关系
    所以commonjs是nodejs的一个规范，它的规范使得nodejs具有的开发服务器的可能。

什么叫核心模块：

commonjs规定的好的功能在nodejs实现了，这样的一些模块我们叫做核心模块。也就是说nodejs中所有文档中给出的api都是核心模块，而我自己写的模块我们称为用户模块

path

该模块包括了一些处理文件路径的功能，可以通过 require(‘path’) 方法来使
用它。

  • path.basename(path)：该方法返回一个路径中最低一级目录名,如果不传入任何参数：直接获取到传入路径中文件的全名称（文件名+扩展名）,如果传入文件对应的后缀，会将后缀名去掉（传错与不传相同）。

  • path.dirname(path)：该方法返回一个路径的目录名

  • path.extname(path):该方法返回路径中的文件扩展名，即路径最低一级的目录中’.’字符后的任何字
    符串。如果路径最低一级的目录中’没有’.’ 或者只有’.’，那么该方法返回一
    个空字符串。

  • path.parse(path)：返回一个数组,将传入的路径进行拆分，将路径的各个部分以对象属性的形式返回

key value
root 根目录
dir 所在目录
base 文件名+扩展名
ext 文件扩展名
name 文件名

  • path.format:将一个路径对象传入，返回一个路径的字符串,注意：win下的路径最好加上反斜杠，max系统下不用关注。

  • path.join：该方法用于合并方法中的各参数并得到一个标准化合并的路径字符串。

  • path.delimiter：得到当前系统的分界符,如果在win系统下为’;’,在其它系统下为’:’

  • isAbsolute:判断一个路径是否是一个绝对路径
  • normalize:将一个非标准路径转换成一个标准路径
  • relative（form路径，to路径）:orm路径到to路径要的查找过程
  • sep：在win系统下得到’\’分隔符，在其它系统下面得到’/‘分隔符
  • posix:所有的路径方法的一个集合

url

  • url.parse(urlString):将一个url转成对象
  • url.format:将一个解析后的URL对象、转成、一个格式化的URL字符串。

queryString

  • querystring.parse():将一个参数字符串转为对象
  • querystring.stringify():将一个对象转为字符串
  • querystring.escape(url):将url中的汉字进行转码
  • querystring.unescape(url):将url中被转码的汉字重新转为汉字
