全局对象和全局变量

Node提供了几个全局对象，他们是所有模块都可以调用的。

  1. globall:表示Node所在的全局环境，类似于window对象，但又不尽相同，比如var x =1在浏览器中等同于window.x=1,在node中的模块文件中声明var x = 1，该变量不是global对象的属性,global.x等于undefined，因为这是该模块全局变量私有的，其他模块无法获取。

    • global.__filename:当前脚本执行所在的全路径(物理路径+文件名)

    • global.__dirname:得到当前文件所在的物理路径

    • global.setTimeout/global.clearTimeout:设置定时器清空定时器

    • global.setInterval/global.clearInterval：设置定时器清除定时器

    • global.setImmediate/clearImmediate：设置定时器清除定时器

    执行顺序测试

  1. process:该对象表示Node所处的当前进程，允许开发者与该模快互动。

    • process.nextTick(callback)：功能是为事件循环设置一项任务,，Node.js 会在
      下次事件循环调响应时调用 callback 。

    • process.abort:立即结束程序运行

    • process.arch:得到当前系统运行的环境

    • process.argv:是命令行参数数组， 第一个元素是 node， 第二个元素是脚本文件名，
      从第三个元素开始每个元素是一个运行参数。

    • process.stdout.write:输出字符串并且不换行输出console.log就是借用这个方法实现的,+\n

    • process.stdin:是标准输出流， 通常我们使用的 console.log() 向标准输出打印
      字符，而 process.stdout.write() 函数提供了更底层的接口。初始时它是被暂停的，要想从标准输入读取数据，你必须恢复流，并手动编写流的事件响应函数。

    • process.stdin.on:当输入内容敲击回车的时候会触发这个事件

    • process.stdin.read:得到输入回车前输入的内容，toString（）以后才能转化字符串
  2. console:指向Node内置的console模块，提供命令行环境中的标准输入,输出功能。
