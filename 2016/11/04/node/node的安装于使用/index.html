nvm# js的模块话

1.0 js的模块化

什么是模块化？，模块化的概念最早是后台中德，随着ajax技术的兴起，js在编程中所占的地位越来越高，同时js的文件也相应的越来越多，为了方便文件的管理和更新，提出了js文件的模块化，将同一类型功能组合在一起。

模块化的演变过程

1.普通的方法书写格式

  • 多人开发引起变量污染

2.使用对象的作用域来解决变量污染问题

  • 如果在这个对象中一些属性不为外界所访问，不能解决

3.使用函数的方法来私有化变量

  • 每次使用都需要创建一个实例

4.使用函数的自执行模式来代替new操作

  • 如果需要扩展

5 使用一个自执行函数在原来的对象上扩展方法

  • 将来自执行的时候要传参数

6.如果要用到第三方模块需要写在自执行函数的参书上

JS模块管理工具

seajs

seajs:第三方模块管理工具
(http://seajs.org)

特点

  1. sea.js遵循cmd规范
  2. node.js一般的书写代码
  3. 依赖自动加载

使用步骤

  • 要定义一个单独的模块:关键字define
  • 要在模版中返回一些方法/属性/对象关键字：exports/module.exports
  • 如果引用一个第三方模块关键字：require
  • 使用seajs定义好的模块关键字：seajs.use

如果要引用两个或者两个以上模块要在seajs.use的第一个参数传进去一个数组即可,后面的回调函数中传入数组中有几个模块传几个参数

seajs遵循的cmd规范
cmd规范的作用：规范明确了模块的的书写方式和基本交互规则

node的特点

node 作为一个服务器语言:

  1. 开启 服务器功能
  2. 读写硬盘数据功能
